# SmartCutElf 优化整改计划

## 目标

基于当前代码实现，优先修复已经确认的稳定性问题、功能失效问题和明显的性能瓶颈，避免建议停留在泛泛层面。

## 已确认问题

### P0 稳定性与功能正确性

1. AI 开关未真正接入处理流程
- 现象：界面中的“字幕+配音”只影响日志显示，不会控制实际字幕和 TTS 逻辑。
- 影响：用户设置与实际行为不一致。
- 涉及文件：
  - `src/ui/main_window.py`
  - `src/core/workflow.py`

2. 预设切换调用了不存在的配置接口
- 现象：`Config.set(..., permanent=True)` 与当前 `Config.set()` 签名不一致。
- 影响：点击“标准/快速”预设可能直接报错。
- 涉及文件：
  - `src/ui/main_window.py`
  - `src/utils/config.py`

3. 历史记录保存调用了不存在的方法
- 现象：界面在处理完成后调用 `db.add_processing_record(...)`，但数据库层未实现该方法。
- 影响：批处理完成后的收尾阶段可能报错。
- 涉及文件：
  - `src/ui/main_window.py`
  - `src/utils/database.py`

4. 批处理时每个视频都会创建独立项目
- 现象：同一轮批量处理没有共享 `project_id`。
- 影响：数据库中的项目粒度错误，历史记录和批处理聚合能力差。
- 涉及文件：
  - `src/core/workflow.py`

5. 文件总大小统计重复累加
- 现象：扫描文件时累加一次，渲染列表时又累加一次。
- 影响：界面显示的总大小不正确。
- 涉及文件：
  - `src/ui/main_window.py`

### P1 性能与并发

6. 高光检测存在重复音频加载和重复视频打开
- 现象：每个分段都会重新读取整段音频，并重复打开视频流。
- 影响：长视频分析耗时明显偏高。
- 涉及文件：
  - `src/core/highlight_detector.py`
  - `src/core/audio_analyzer.py`
  - `src/core/video_analyzer.py`

7. 合并列表文件名固定，存在并发冲突风险
- 现象：输出目录统一写入 `concat_list.txt`。
- 影响：多任务同时合并时可能互相覆盖。
- 涉及文件：
  - `src/core/video_processor.py`

8. FFmpeg 池为全局单例，后续配置可能不生效
- 现象：第一次初始化后，新的并发配置不会重新建池。
- 影响：运行时配置与实际执行不一致。
- 涉及文件：
  - `src/utils/ffmpeg_pool.py`
  - `src/core/video_processor.py`

### P2 可维护性

9. 日志与调试输出不统一
- 现象：存在 `print("DEBUG: ...")` 和无效的 `stdout` 重定向。
- 影响：日志行为不一致，后续排障成本高。
- 涉及文件：
  - `src/ai/speech_recognition.py`
  - `src/ui/main_window.py`

10. SQLite 并发配置偏保守
- 现象：每次新建连接，但未显式设置 WAL 和 busy timeout。
- 影响：批处理场景下更容易遇到锁等待问题。
- 涉及文件：
  - `src/utils/database.py`

## 执行顺序

### 第一批
- 修复 AI 开关接线问题
- 修复预设配置接口不匹配
- 修复历史记录保存路径
- 修复文件大小统计错误

### 第二批
- 调整批处理项目模型
- 优化高光分析中的重复 I/O
- 修复合并列表临时文件冲突

### 第三批
- 清理调试输出与无效重定向
- 视情况增强 SQLite 并发配置
- 复查 FFmpeg 并发池行为

## 验收标准

- UI 配置与实际处理结果一致
- 批量处理完成后不再因历史记录逻辑报错
- 同一批次视频归属同一项目
- 长视频分析阶段不再出现明显的重复读取开销
- 多任务并发时不因固定临时文件名相互干扰
